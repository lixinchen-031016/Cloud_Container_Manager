# Cloud Container Manager - 使用指南

## 📖 快速开始

### 1. 安装依赖

```bash
cd /Users/lixinchen/codeProject
pip install -r requirements.txt
```

### 2. 运行应用

```bash
python src/main.py
```

### 3. 连接服务器

1. 点击左侧边栏的 **"+ 添加服务器"** 按钮
2. 选择认证方式：
   - **密码认证**：输入主机、端口、用户名和密码
   - **密钥认证**：输入主机、端口、用户名，选择 SSH 私钥文件
3. 点击 **"测试连接"** 验证配置
4. 点击 **"连接"** 建立连接

### 4. 查看监控数据

连接成功后，切换到 **"📊 服务器监控"** 标签页：
- **CPU**: 显示使用率、核心数和型号
- **内存**: 显示使用率和详细使用情况
- **系统负载**: 显示 1/5/15 分钟平均负载
- **网络**: 显示各网卡的流量统计
- **磁盘**: 显示各分区的使用情况

### 5. 管理 Docker 容器

切换到 **"🐳 容器管理"** 标签页：
- 查看所有容器列表（包括已停止的）
- **右键点击容器** 可以进行以下操作：
  - 查看日志（显示最近 200 行）
  - 启动/停止/重启容器
  - 删除容器

## 🔧 功能详解

### SSH 连接

支持两种认证方式：

#### 密码认证
适用于大多数 Linux 服务器：
- 主机：IP 地址或域名
- 端口：默认 22
- 用户名：通常是 root 或其他用户
- 密码：用户登录密码

#### 密钥认证
更安全的认证方式：
1. 生成 SSH 密钥对（如果还没有）：
   ```bash
   ssh-keygen -t rsa -b 4096
   ```
2. 将公钥复制到服务器：
   ```bash
   ssh-copy-id user@hostname
   ```
3. 在应用中选择私钥文件（通常是 `~/.ssh/id_rsa`）

### 服务器监控

监控数据每 2 秒自动刷新一次：

- **CPU 使用率**: 通过 `top` 命令获取
- **内存信息**: 通过 `free -k` 命令获取
- **磁盘空间**: 通过 `df -k` 命令获取
- **网络流量**: 通过 `/proc/net/dev` 获取累计流量
- **系统负载**: 通过 `uptime` 命令获取

### 容器管理

容器操作说明：

| 操作 | 说明 |
|------|------|
| 查看日志 | 显示容器最近 200 行日志 |
| 启动 | 启动已停止的容器 |
| 停止 | 优雅地停止运行中的容器 |
| 重启 | 重启容器 |
| 删除 | 删除容器（强制删除已停止的容器） |

## ⚙️ 高级配置

### 自定义刷新频率

修改 `src/ui/server_monitor.py` 中的定时器间隔：

```python
# 在 MonitorWorker.run() 方法中
self.msleep(2000)  # 2000ms = 2 秒，可改为其他值
```

### 修改默认端口

在连接对话框中，端口默认为 22。如果使用非标准 SSH 端口，直接在连接时指定即可。

### 多服务器管理

当前版本支持依次连接多个服务器（每次连接一个）。如需同时管理多个服务器，可以：
1. 保存常用服务器配置（待实现）
2. 快速切换不同的服务器连接

## 🐛 故障排除

### 无法连接服务器

**问题**: 连接超时或被拒绝

**解决方法**:
1. 检查服务器 IP 地址是否正确
2. 确认 SSH 服务正在运行：`sudo systemctl status sshd`
3. 检查防火墙设置：`sudo ufw status`
4. 确认端口 22 开放

### Docker 命令执行失败

**问题**: "docker: command not found"

**解决方法**:
1. 在服务器上安装 Docker
2. 将用户添加到 docker 组：
   ```bash
   sudo usermod -aG docker $USER
   ```
3. 重新登录使更改生效

### 权限错误

**问题**: 执行 Docker 命令时提示权限不足

**解决方法**:
1. 使用 sudo 执行 Docker 命令（需要配置免密 sudo）
2. 或者将用户添加到 docker 组（推荐）

### 日志显示乱码

**问题**: 容器日志显示乱码

**解决方法**:
1. 确保容器使用 UTF-8 编码
2. 在 Dockerfile 中设置：
   ```dockerfile
   ENV LANG C.UTF-8
   ```

### 线程模型

为避免 UI 阻塞，应用使用多线程：

- **主线程**: UI 渲染和用户交互
- **工作线程**: SSH 连接、命令执行、数据获取
- **信号槽机制**: 线程间安全通信

### 安全考虑

1. **密码处理**: 密码不在日志中显示
2. **密钥文件**: 验证密钥文件路径和权限
3. **主机验证**: 自动接受未知主机密钥（生产环境应谨慎）
4. **操作确认**: 危险操作（如删除容器）需要确认
